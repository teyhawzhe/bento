## Why

最新 A006 已新增「查詢供應商清單及其對應的便當選項」需求，並明確列出 `GET /api/admin/suppliers` 作為管理員查詢能力的一部分。現況前端雖然已經做了部分供應商與菜單的關聯呈現，但主要仍靠既有 `/suppliers` 與 `/menus` 自行拼接，尚未對齊新的 A006 API 契約與管理員查詢語意，因此需要補一個同步 change。

## What Changes

- 對齊 A006，新增管理員查詢「供應商清單及其對應便當選項」的明確能力與 API 契約。
- 讓供應商管理頁直接使用供應商與其便當選項的查詢結果，而不是只靠分離的 `suppliers` 與 `menus` 清單自行拼接。
- 將供應商管理區調整為 `供應商` / `菜單` 兩個工作分頁，讓資料主檔維護與菜單維護有清楚分工。
- 調整菜單管理顯示方式與建立菜單流程，讓供應商辨識與供應商上下文操作更自然，切換分頁時也不會失去目前正在處理的供應商。
- 保持既有供應商 CRUD 與菜單 CRUD 能力不變，本 change 主要補齊 A006 新查詢需求與相關前端體驗。

## Capabilities

### New Capabilities

None.

### Modified Capabilities

- `admin-order-management`: 管理員需能查詢供應商清單及其對應的便當選項，對齊 A006 的新 API 與查詢流程。
- `bento-supplier-management`: 供應商管理頁需能呈現所選供應商底下的菜單資訊，並以新的管理員供應商查詢能力作為資料來源；同時需在供應商工作分頁內清楚區分新增與編輯供應商主檔。
- `bento-menu-management`: 菜單清單與建立/編輯流程需更清楚顯示菜單所屬供應商，降低只靠 `supplierId` 辨識的負擔，並獨立在菜單工作分頁中維護。

## Impact

- 受影響程式碼：backend 管理員供應商查詢 controller/service/dto、frontend 供應商管理與菜單管理 UI、相關 state 與操作流。
- 受影響 API：新增或調整 `GET /api/admin/suppliers`，讓回應直接包含供應商對應便當選項。
- 受影響體驗：管理員可直接查詢供應商及其便當選項，並在 `供應商` / `菜單` 分頁中維持清楚的操作脈絡，更直覺地回答「這家便當是誰的」。
