## Why

目前前端把多個員工與管理員功能堆疊在同一個長頁面中，登入後雖然能操作既有功能，但切換成本高，也缺少一致的互動節奏。A009 要補上角色導向的 TAB 分頁與統一 MessageBox 提示規範，讓使用者在不重整整頁的情況下切換功能，並用一致的成功、錯誤、警告與刪除確認彈窗完成操作回饋。

## What Changes

- 新增前端 TAB 導航能力，讓登入後的員工與管理員依角色看到對應功能頁籤。
- 定義員工端 TAB 為「訂便當」與「修改密碼」，其中「訂便當」TAB 左側顯示下單流程、右側同步顯示「我的訂單」清單。
- 定義管理員端 TAB 為「訂單管理」、「供應商管理」、「報表設定」與「系統設定」，並以頁籤切換承載既有 A006、A007、A008、A004 功能。
- 移除登入後的「員工入口 / 管理員入口」選擇頁，改為依登入回傳角色直接導向對應 TAB 主頁。
- 新增統一的 MessageBox UI 規範，涵蓋操作成功、操作失敗、表單驗證失敗與刪除確認四種情境。
- 將現有前端中的行內成功/錯誤訊息與瀏覽器原生確認流程，收斂為由 MessageBox 控制的互動模式。

## Capabilities

### New Capabilities
- `frontend-tab-navigation-and-message-box`: 定義登入後依角色直接導向對應 TAB 主頁、顯示固定頁籤、切換對應功能內容，並以統一 MessageBox 顯示成功、錯誤、警告與確認提示。

## Impact

- 受影響程式碼：`frontend/src/App.tsx` 的登入流程、登入後版面、各功能區塊切換方式、成功/錯誤訊息處理與刪除確認流程。
- 受影響元件：需新增或抽離前端共用 TAB 與 MessageBox 元件，並整理現有員工/管理員畫面分區。
- 受影響 API：不新增後端 API，但前端需整合既有員工與管理員登入入口，並統一既有 A002、A004、A006、A007、A008 API 的成功與失敗回饋方式。
- 相依項目：沿用 A001 登入與角色資訊、A002 員工訂餐、A004 錯誤通知信箱設定、A006 管理員訂單管理、A007 供應商管理、A008 報表收件設定。
