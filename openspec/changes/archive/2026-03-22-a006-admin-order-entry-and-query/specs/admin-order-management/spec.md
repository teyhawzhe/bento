## ADDED Requirements

### Requirement: 管理員可以查詢所有員工的訂餐記錄
系統 SHALL 允許已驗證的管理員查詢所有員工的訂餐記錄，並支援以訂餐日期與員工編號作為篩選條件。查詢結果 MUST 回傳管理頁顯示所需的訂單資訊，至少包含訂餐日期、員工姓名、便當名稱，以及建立者資訊。

#### Scenario: 管理員依日期查詢訂餐記錄
- **WHEN** 管理員以指定日期呼叫查詢所有員工訂餐記錄功能
- **THEN** 系統回傳該日期符合條件的訂單清單，且每筆資料包含員工姓名與便當名稱

#### Scenario: 管理員依員工篩選訂餐記錄
- **WHEN** 管理員以指定員工編號與日期呼叫查詢所有員工訂餐記錄功能
- **THEN** 系統僅回傳該員工在指定條件下的訂單資料

### Requirement: 管理員可以在截止前代替員工新增隔日訂餐
系統 SHALL 允許已驗證的管理員代替指定員工建立隔日訂餐。系統 MUST 在建立前驗證目前時間早於該訂餐日前一日 17:00，且成功建立時 MUST 將 `orders.created_by` 記錄為實際操作的管理員員工編號。

#### Scenario: 管理員在截止前成功代訂隔日便當
- **WHEN** 管理員在訂餐日前一日 17:00 前為指定員工建立隔日訂單
- **THEN** 系統建立訂單成功，並將 `created_by` 記錄為該管理員

#### Scenario: 管理員超過截止時間無法代訂
- **WHEN** 管理員在訂餐日前一日 17:00 後為指定員工建立隔日訂單
- **THEN** 系統拒絕請求並提示已超過代訂截止時間

#### Scenario: 管理員建立非隔日訂單時被拒絕
- **WHEN** 管理員嘗試為指定員工建立非隔日日期的訂單
- **THEN** 系統拒絕請求並提示僅允許建立隔日訂單
