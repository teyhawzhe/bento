## MODIFIED Requirements

### Requirement: 管理員可以在截止前代替員工新增隔日訂餐
系統 SHALL 允許已驗證的管理員代替指定員工建立隔日訂餐。系統 MUST 在建立前驗證目前時間早於該訂餐日前一日 16:30，且成功建立時 MUST 將 `orders.created_by` 記錄為實際操作的管理員員工編號。

#### Scenario: 管理員在截止前成功代訂隔日便當
- **WHEN** 管理員在訂餐日前一日 16:30 前為指定員工建立隔日訂單
- **THEN** 系統建立訂單成功，並將 `created_by` 記錄為該管理員

#### Scenario: 管理員超過截止時間無法代訂
- **WHEN** 管理員在訂餐日前一日 16:30 後為指定員工建立隔日訂單
- **THEN** 系統拒絕請求並提示已超過代訂截止時間

#### Scenario: 管理員建立非隔日訂單時被拒絕
- **WHEN** 管理員嘗試為指定員工建立非隔日日期的訂單
- **THEN** 系統拒絕請求並提示僅允許建立隔日訂單
